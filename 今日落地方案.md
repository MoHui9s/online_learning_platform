## 六、技术简化方案

### 前端简化策略

1. **统一使用 Element Plus 现成组件**
   - Table（列表）
   - Card（卡片）
   - Collapse（折叠面板）
   - Pagination（分页）
   - Form（表单）
   - Dialog（弹窗）

2. **不做复杂交互**
   - 不做拖拽排序（章节顺序由后端返回）
   - 不做实时搜索（按钮提交）
   - 不做虚拟滚动（列表限制 20 条/页）

3. **视频播放器最小配置**
   ```js
   // 只用 video.js 基础功能，不做自定义皮肤
   videojs('my-video', {
     controls: true,
     sources: [{ src: videoUrl, type: 'video/mp4' }]
   })
   ```

4. **ECharts 使用官方示例**
   - 复制官方 Calendar / Bar / Radar 示例代码
   - 只改数据源，不改样式

### 后端简化策略

1. **不做权限细化**
   - teacher 可操作全部课程（不校验 teacher_id）
   - admin 可操作全部数据

2. **不做文件上传**
   - 课程封面/视频 URL 直接填外链或本地路径
   - 演示时用测试视频（BigBuckBunny.mp4）

3. **不启动 ChromaDB**
   - RAG 功能演示时手动调 API（Postman）
   - 前端不做 SSE 流式页面
